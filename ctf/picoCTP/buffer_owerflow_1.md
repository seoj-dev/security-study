#Nice netcat(picoCTF)
- 유형 : general
- 날짜 : 2026-02-06

##처음 접근
- powershell에 바로 명령어 입력

##막힌 부분
- nc명령어가 없다는 오류 메세지가 뜸
- nc명령어의 개념을 정확히 몰랐음

##핵심포인트
- 리눅스에서 ns 명령어로 서버와 통신
- 서버로부터 받은 숫자를 아스키코드로 전환해 flag값을 얻을 것
- (이떄 파이썬을 이용하여 많은 숫자를 한 번에 아스키코드로 변환)

##알게된 것
- nc(netcat) : 중간자의 간섭없이 서버와 바로 통신할 수 있음
- nc [주소] [포트]

#where are the robots
- 유형 : web
- 날짜 : 2026-02-06

##처음 접근
- 안 보이는 내용을 찾는 것이니 html 코드역역을 살펴봄

##막힌 부분
- 해당 웹사이트의 코드 및 헤더를 다 살펴보아도 robot과 관련된 내용을 찾지 못함

##핵심포인트
- 여기서 robots는 robots.txt를 의미
- url에 /robots.txt를 입력하여 정보를 얻음

##알게된 것
- robots. txt : 웹사이트 루트에 있는 규칙 파일
- 트롤러 트래픽을 관리하기 위해 사용
